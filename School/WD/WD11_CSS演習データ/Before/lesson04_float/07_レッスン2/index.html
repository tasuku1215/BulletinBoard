BOXのレイアウト

BOXの構造を理解する

BOXの構造は、文字や画像といった「コンテンツを表示する領域」、 コンテンツとボーダーの間の余白の「パディング」、パディングの枠線である「ボーダー」、 枠線の外側の余白の「マージン」によって構成されている。 これら「コンテンツを表示する領域」「パディング」「ボーダー」「マージン」の領域全体を「ボックス」という。 このボックスの領域や枠線に対して、それぞれのスタイルを指定することで要素（ボックス）をデザインしていきます。 また、このボックスを元にWebページ全体のレイアウト（デザイン）を考えることを「ボックスモデルデザイン」といいます。

要素をグループ化する

要素の特性

ＨＴＭＬテキストの本文を構成する要素には、大きく分けてブロックレベル要素と、インライン要素の２つがある。

ブロックレベル要素
「ひとつの独立したまとまり」という意味。
ブロックレベル要素同士は改行されて表示する。
ブロックレベル要素タグ（抜粋）
要素名 説明
div 範囲を指定してグループ化する（ブロックレベル）
h1～h6 見出しを指定
p 段落
ul 順序無しリスト（黒丸がつく）
ol 順序有リスト（番号がつく）
dl 定義型リスト
table 表を指定
address 連絡先・制作者情報など
インライン要素
「行内の一部」という意味。
１行の中にテキストと一緒に並べて配置できる小さなまとまりを指します。 ブロックレベル要素は、インライン要素を含むことができますが、 インライン要素がブロックレベル要素を含むことはできない。
インライン要素タグ（抜粋）
要素名 説明
span 範囲を指定してグループ化する（インライン）
em 強調
strong より強い強調
a リンク／アンカー
img 画像の表示
br 改行
sup 上付き文字
sub 下付き文字

要素のグループ化する為のタグ

<div>と<span>を使って内容をグループ化する事ができる。

<div>～</div>　→　ブロックレベル要素

<span>～</span>　→　インライン要素やテキスト

<div>
<h2>見出しレベル２</h2>
<p>指定した箇所だけ<span>赤色</span>にする。</p>
</div>

また、場合によってはそれぞれの要素に名前をつけて識別する必要がある。

理由：CSSでは、<div>や<span>要素をターゲットに指示をするが、 グループが複数あると識別できないため、全て同じデザインになってします。

HTMLでの名前の付け方と、CSSでの指示の仕方

<開始タグ id="任意の名前">

idタグは、ページ内で間違いなく１回しか出てこない部分を指定。

CSSでの記述方法　→　#任意の名前
任意でつけた名前の前に#（シャープ）を付ける！
ID名によく使用される名前
ID名 説明
wrapper body要素の中に入っている要素をすべて囲み、まとめてデザインするときによく使われる。 containerと名前をつける場合もある。
header HTML文書上部の、Webサイトで統一して出てくる要素を囲み、まとめてデザインするときによく使用される。
単にはじめの要素を囲うときに使用される。
content HTML文書のコンテンツ部分の要素を囲み、まとめてデザインするときによく使われる。mainと名づけられることもある。
footer HTML文書下部の、Webサイトで統一して出てくる要素を囲み、まとめてデザインするときによく使われる。
nav ナビゲーションをデザインするときによく使われる。menuと名づけられることもある。
sidebar HTML文書のコンテンツ部分に付属した要素。
subと名づけられることもある。

<開始タグ class="任意の名前">

classタグは、１ページ内で同じ名前を何度も使用できる。

CSSでの記述方法　→　.任意の名前
任意でつけた名前の前に.（ドット）を付ける！

ソース記述例

HTMLの記述例

<div id="contents">
<h2>見出しレベル２</h2>
<p>指定した箇所だけ<span="red">赤色</span>にする。</p>
</div>

CSSの記述例

#contents {
　　　padding: 20px;
　　　margin: 20px;
　　　border: solid 1px #000;
}
.red {
　　　color: #f00;
}
